# Superpowers Workflow

This directory is reserved for active Superpowers plans and specs only.

- Start each task with `superpowers:using-superpowers` and load any matching skill before editing.
- Code, refactors, and bug fixes follow `superpowers:test-driven-development`.
- TDD uses Red-Green-Refactor: RED writes and runs a failing test, GREEN adds the smallest implementation, and REFACTOR cleans up while tests stay green.
- Before reporting completion, run `superpowers:verification-before-completion` and record fresh command evidence in the handoff.
- Do not keep completed dated plans or screenshots as repository history.
- Create new plans under `docs/superpowers/plans/` only while executing active work.
- Move durable architecture decisions to `docs/architecture/`.
- Keep temporary reviews, browser screenshots, and test artifacts in ignored local paths.
