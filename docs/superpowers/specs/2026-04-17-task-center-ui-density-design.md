# Task Center UI 密度与工作台重构设计

## 1. 背景

当前 WebUI 已从脚本配置后台演进为签到任务中心，但视觉和交互仍偏“大卡片管理台”。  
主要问题：

- 组件整体偏大，桌面端空间利用率不足
- 首页更像概览拼盘，不像任务工作台
- `站点`、`账号`、`今日任务` 仍以卡片流为主，批量扫描效率不高
- 高级配置虽然已降级，但仍持续占据注意力

本次设计不改业务流程，只重构前端信息密度、页面层级和关键页面布局。

## 2. 目标

将 WebUI 明确收敛为“多站点多账号 new-api 自动签到任务工作台”。

设计目标：

- 桌面端优先，提高单位屏幕信息密度
- 保留呼吸感，不做拥挤后台
- 强化“今日任务 -> 异常处理 -> 历史收益”主工作流
- 将高频页面改为更适合运营使用的紧凑布局

成功标准：

- 主导航、页头、摘要区、卡片、表单统一缩小一档
- `今日任务` 成为最强主线页面
- `站点`、`账号` 从重卡片布局向紧凑列表或表格布局收敛
- 高级配置明显降权，但仍可访问

## 3. 已确认的视觉方向

用户已确认使用：

- `A 为主 + B 的导航/表格 + C 的间距`

对应解释：

- 主基调：Balanced Compact
- 导航和表格：吸收 Dense Dashboard 的高密特征
- 间距节奏：保留 Airy Minimal 的呼吸感

终端优先级：

- 两端同时优化，但以桌面端为主

## 4. 设计原则

### 4.1 信息架构原则

- 首页只回答今日状态，不承载过多配置态信息
- 今日任务页承担日常执行主入口
- 异常页承担诊断与补救入口
- 站点页和账号页承担资产管理职责
- 高级配置保留在次级层，不与主任务流竞争

### 4.2 密度原则

- 缩小导航项高度、页头高度、按钮高度、卡片留白
- 提高列表信息密度，减少纵向滚动
- 状态信息优先横向聚合，不重复堆叠说明文
- 空状态和帮助文案保留，但收短

### 4.3 视觉原则

- 延续现有深色侧栏 + 浅色工作区的视觉语言
- 不重做品牌色，不改变主色基调
- 减少过大的圆角、过厚的卡片感和过强的展示型统计块
- 不引入复杂动画，只保留轻量 hover 和状态反馈

## 5. 页面级重构方案

### 5.1 App Shell

- 缩窄左侧导航宽度
- 降低导航项高度与图标容器尺寸
- 压缩顶部栏高度
- 高级配置区继续保留在侧栏下部，但视觉弱化
- `当前页面` 与 `会话信息` 继续保留，压缩为更轻的摘要块

### 5.2 首页

定位为“今日战情页”，不是系统概览页。

调整：

- 第一屏只保留今日关键指标：
  - 今日总任务
  - 待处理
  - 成功
  - 失败/阻塞
  - 今日额度
- 部署模式、本地调度等技术态信息降为次级信息
- 主体改成两列：
  - 左：今日任务快照
  - 右：异常摘要 / 最近异常
- 站点概览和账号动态保留，但降低视觉优先级

### 5.3 今日任务

定位为主工作台。

调整：

- 顶部操作区保留四个高频动作：
  - 导入
  - 生成
  - 执行
  - 刷新
- 摘要区压缩成一行紧凑状态条
- 筛选器改为更紧凑的四格筛选区
- 任务列表从大卡片流收敛为紧凑列表或表格式布局
- 每条任务保留最核心字段：
  - 站点
  - 账号
  - 状态
  - 执行链
  - 时间
  - 奖励额度
  - 快捷动作

### 5.4 站点

定位为站点资产管理。

调整：

- 左侧录入表单保留
- 右侧站点清单改为高密度列表表达
- 每个站点只保留关键状态：
  - 启用状态
  - 兼容等级
  - 账号数
  - 今日完成
  - 今日异常
  - 签到能力
- 弱化过多 badge 堆叠

### 5.5 账号

定位为执行主体资产管理。

调整：

- 左侧账号表单保留
- 右侧清单向高密度列表靠拢
- 重点展示：
  - 所属站点
  - 账号标识
  - 认证方式
  - 最近签到状态
  - 累计签到
  - 累计额度

### 5.6 异常处理

定位为人工补救入口。

调整：

- 保持简洁风格
- 异常列表优先突出：
  - 异常类型
  - 账号
  - 站点
  - 最近发生时间
  - 当前状态
- 空状态保留，但压缩占高

## 6. 全局组件与 Token 调整

### 6.1 Spacing

- 建立更紧的全局间距层级
- 优先使用 `8 / 12 / 16 / 20 / 24` 节奏
- 摘要条、筛选区、卡片内部间距统一收紧

### 6.2 尺寸

- 侧栏宽度缩小
- 顶栏高度下降
- 按钮高度从当前大号尺寸统一下调一档
- 输入框与按钮对齐
- Badge 和状态胶囊统一缩小

### 6.3 卡片

- 降低圆角和 padding
- 降低展示型阴影存在感
- 减少纯展示卡数量，把信息更多放进紧凑列表

## 7. 非目标

- 不改后端接口和业务主链路
- 不重构部署模式、调度模式和配置模型
- 不改品牌配色体系
- 不做大幅动画和装饰性视觉实验
- 不把全部页面改成无留白纯表格后台

## 8. 验证标准

设计落地后需满足：

- 桌面端首页首屏能同时看到主摘要和核心任务区
- 今日任务页在常见桌面分辨率下显著减少滚动
- 站点页、账号页同屏可见信息量明显提高
- 侧栏、页头、按钮、筛选器视觉尺寸明显缩小
- 移动端导航与任务页同步收紧，但信息架构不重写

## 9. 实施边界

本规格只定义：

- 视觉密度目标
- 页面级布局方向
- 组件压缩规则
- 非目标和验收口径

后续实现计划应按本规格拆成：

- 全局 Shell 与 Token
- 首页与今日任务
- 站点与账号
- 异常与报表补齐
